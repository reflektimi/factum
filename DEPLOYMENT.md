# Factum Deployment Guide

This guide covers deploying the **Factum** financial management application to production using **Neon** (PostgreSQL) and **Render** (application hosting).

## Architecture Overview

- **Database**: Neon PostgreSQL (Serverless)
- **Backend**: Render Web Service (Docker)
- **Frontend**: Inertia.js (served with backend)
- **File Storage**: Public storage (served via Laravel)

---

## Prerequisites

1. **GitHub Account** - Code repository
2. **Neon Account** - Database hosting ([neon.tech](https://neon.tech))
3. **Render Account** - Application hosting ([render.com](https://render.com))

---

## Step 1: Set Up Database (Neon)

### 1.1 Create Neon Project
1. Go to [console.neon.tech](https://console.neon.tech)
2. Click **"New Project"**
3. Configure project:
   - **Name**: `factum-production`
   - **Region**: Choose closest to your users
   - **PostgreSQL Version**: 16 (latest)
4. Click **"Create Project"**

### 1.2 Get Database Credentials
After creation, you'll see connection details:
```
Host: ep-xxx-xxx.us-east-2.aws.neon.tech
Database: neondb
User: neondb_owner
Password: [generated-password]
Port: 5432
```

**Save these credentials** - you'll need them for Render configuration.

### 1.3 Connection String Format
```
postgresql://[user]:[password]@[host]/[database]?sslmode=require
```

## Step 2: Prepare Application for Deployment

### 2.1 Update Environment Variables
The `render.yaml` is already configured, but ensure these are set in Render dashboard:

**Required:**
- `APP_NAME=Factum`
- `APP_ENV=production`
- `APP_DEBUG=false`
- `APP_URL=https://your-app.onrender.com`
- `APP_KEY=` (auto-generated by Render)

**Database (from Neon):**
- `DB_CONNECTION=pgsql`
- `DB_HOST=[neon-host]`
- `DB_PORT=5432`
- `DB_DATABASE=[neon-database]`
- `DB_USERNAME=[neon-user]`
- `DB_PASSWORD=[neon-password]`

**Optional (if using Redis/Upstash):**
- `REDIS_HOST`
- `REDIS_PASSWORD`
- `REDIS_PORT`

### 2.2 Verify Dockerfile
The existing `Dockerfile` is configured for production deployment with:
- PHP 8.2 with required extensions
- PostgreSQL support (`pdo_pgsql`)
- Composer dependencies (optimized, no dev packages)
- NPM build for Inertia.js assets

## Step 3: Deploy to Render

### 3.1 Connect GitHub Repository
1. Go to [dashboard.render.com](https://dashboard.render.com)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub account
4. Select repository: `L0t1/finances-saas`

### 3.2 Configure Web Service
**Basic Settings:**
- **Name**: `factum-app`
- **Region**: Same as Neon database
- **Branch**: `main`
- **Runtime**: `Docker`
- **Dockerfile Path**: `./Dockerfile`

**Build Settings:**
- Build Command: _(leave empty, Docker handles this)_
- Start Command: `./docker/startup.sh`

**Instance Type:**
- Free tier: **Starter** (512 MB RAM)
- Recommended: **Starter Plus** ($7/mo, 1 GB RAM, faster)

### 3.3 Set Environment Variables
In Render dashboard, add all environment variables from section 2.1:

1. Navigate to **Environment** tab
2. Add **Database credentials** from Neon
3. Add **APP_KEY**: Click "Generate" button
4. Add **APP_URL**: `https://factum-app.onrender.com` (or your custom domain)

### 3.4 Deploy
1. Click **"Create Web Service"**
2. Render will:
   - Clone your repository
   - Build Docker image
   - Run migrations via `startup.sh`
   - Start the application

**Deployment takes 5-10 minutes** on first deploy.

## Step 4: Post-Deployment Tasks

### 4.1 Run Database Seeder (Optional)
To populate demo data:

```bash
# In Render Shell (go to "Shell" tab in dashboard):
php artisan migrate:fresh --seed
```

**⚠️ WARNING**: This will clear existing data. Only run on first deployment.

### 4.2 Generate Application Key (if not auto-generated)
```bash
php artisan key:generate --show
```
Copy the key and add to environment variables.

### 4.3 Clear Caches
```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### 4.4 Test the Application
1. Visit: `https://factum-app.onrender.com`
2. Login with: `admin@finances.com` / `password`
3. Verify:
   - Dashboard loads with data
   - Logo displays correctly
   - Pagination works
   - "Refresh Data" button functions

---

## Step 5: Custom Domain (Optional)

### 5.1 Add Domain in Render
1. Go to **Settings** → **Custom Domains**
2. Click **"Add Custom Domain"**
3. Enter: `app.factum.com` (or your domain)

### 5.2 Configure DNS
Add CNAME record in your DNS provider:
```
CNAME  app  factum-app.onrender.com
```

### 5.3 SSL Certificate
Render automatically provisions SSL via Let's Encrypt.

## Troubleshooting

### Issue: "500 Server Error"
**Solution**: Check logs in Render dashboard
```bash
# Common fixes:
php artisan storage:link
php artisan config:clear
```

### Issue: "Database connection failed"
**Solution**: Verify Neon credentials in environment variables
- Ensure `sslmode=require` is set
- Check Neon database is active (not suspended)

### Issue: Assets not loading
**Solution**:
```bash
npm run build
php artisan config:clear
```

### Issue: Migrations fail
**Solution**:
```bash
# SSH into Render shell
php artisan migrate:fresh
```

## Monitoring & Maintenance

### View Logs
```bash
# In Render Shell
tail -f storage/logs/laravel.log
```

### Database Backups (Neon)
Neon provides automatic daily backups for 7 days (free tier).

To create manual backup:
1. Go to Neon Console
2. Select project
3. Click **"Backups"** → **"Create Backup"**

### Performance Optimization
1. **Enable caching**:
   ```bash
   php artisan config:cache
   php artisan route:cache
   php artisan view:cache
   ```

2. **Use Redis for sessions (Upstash)**:
   - Create Upstash Redis instance
   - Update `SESSION_DRIVER=redis` in env vars
